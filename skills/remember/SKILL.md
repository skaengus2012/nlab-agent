# Remember

## Goal
Promote durable, reusable knowledge into project memory or documentation.

## Workflow
1. **Gather**: Identify important decisions, facts, or conventions discovered in the session.
2. **Classify**:
   - Durable project fact (e.g., "We use Coil for image loading")
   - Temporary working note (e.g., "The bug is only on API 28")
   - Operator preference
3. **Store**: Write the information to the appropriate location (e.g., a `MEMO.md` or updating `README.md`).

## Guidelines
- Keep entries concise and actionable.
- Prefer project-level documentation for long-term knowledge.
- Call out and resolve any conflicting information discovered.
